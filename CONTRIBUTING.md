# Contributing to Fusion 360 CAM AI Assistant

Thank you for your interest in contributing! This project aims to make CAM workflows smarter and more accessible.

## Ways to Contribute

### 1. Testing and Bug Reports

- Test with different machine types (3-axis, 4-axis, 5-axis)
- Test with various materials and geometries
- Report issues with detailed reproduction steps

### 2. CAM Strategy Rules

Help improve suggestions by contributing:
- Material-specific cutting parameters
- Machine-specific constraints
- Industry best practices for different operations

### 3. Tool Libraries

Contribute tool library definitions:
- Your shop's tool collection
- Vendor-specific tool data
- Material-specific presets

### 4. Documentation

- Usage examples and tutorials
- Video walkthroughs
- Translations

### 5. Code Contributions

- New CAM operations
- Improved geometry analysis
- Learning algorithm enhancements
- Performance optimizations

## Development Setup

1. Fork and clone the repository
2. Install Fusion 360 and MCP-Link Server
3. Load the add-in in Fusion 360
4. Make changes and test

## Code Style

- Python code follows PEP 8
- Use descriptive variable names
- Add docstrings to functions
- Include type hints where helpful

## Submitting Changes

1. Create a feature branch: `git checkout -b feature/your-feature`
2. Make your changes
3. Test thoroughly in Fusion 360
4. Commit with clear messages
5. Push and create a Pull Request

## Pull Request Guidelines

- Describe what the PR does and why
- Include test results or screenshots
- Reference any related issues
- Keep PRs focused on a single change

## Reporting Issues

Include:
- Fusion 360 version
- Operating system
- Steps to reproduce
- Expected vs actual behavior
- Error messages or logs

## Code of Conduct

- Be respectful and constructive
- Help newcomers learn
- Focus on the technical merits
- Credit others' contributions

## Questions?

Open a Discussion on GitHub or reach out to the maintainers.

Thank you for helping make CAM workflows better!
